  • Para los datos de validacin seguiremos una serie de pasos:
  1. Generar XTest de forma similar a como se gener X, pero usando solo los datos de test.
  2. Estimar el valor de $Y$ para los datos de validacin.
  3. Generar YTest de forma similar a como se gener Y.
  4. Calcular RSSTest.
  5. Calcular TSSTest, estimando el promedio de YTest con la funcin mean de numpy.
  6. Obtener el valor de nTest, la cantidad de observaciones de los datos de validacin (debe tener un valor distinto a n, pues hay ms sujetos en el set de entrenamiento que en el de validacin).
  7. Obtener el valor de mTest, la cantidad de variables en los datos de validacin (debe tener el mismo valor que m, pues usamos las mismas variables para entrenar y validar).
  8. Calcular RSETest (residual standard error en validacin).
  9. Calcular R2Test (R cuadrada en validacin).
In[142]:
# Genera el elemento XTest
XTest = test.drop('presion', axis = 1)

# Estima los valores de Y para los datos de validacin
yhatTest = results.predict(sm.add_constant(XTest))

# Genera el elemento YTest
YTest = test.presion

# Calcula el RSS de validacin
RSSTest = sum((YTest-yhatTest)**2)

# Calcula el TSS de validacin
TSSTest = sum((YTest-np.mean(YTest))**2)

# Define el valor de n para los datos de prueba
nTest = XTest.shape[0]

# Define el valor de m para los datos de prueba
mTest = XTest.shape[1]

# Calcula el RSE de validacin
RSETest = np.sqrt(RSSTest/(nTest-mTest-1))

# Calcula el R^2 de validacin
R2Test = 1 - RSSTest / TSSTest

# Imprime el RSE
print("RSE =", RSETest)

# Imprime el R^2
print("R^2 =", R2Test)

Idealmente, si el modelo fuera perfecto, veramos solo la lnea roja, que representara que los valores reales y el valor estimado fueron idnticos. En este caso, no fue as; sin embargo, se nota una tendencia de los datos a agruparse cerca de la linea roja.

El modelo presenta un margen de error considerable, ya que los puntos no estn muy cerca de la lnea roja que representa la igualdad entre los valores reales y estimados. Idealmente, si el modelo fuera perfecto, se tendra una lnea recta con una pendiente de (45 grados), donde los valores reales y estimados seran idnticos.

En este caso, la tendencia general es visible, pero los valores predichos no se alinean completamente con los reales. Esto se refleja en un RSE de 4.72 y un R de 0.54, indicando que el modelo explica solo la mitad de la variabilidad en los datos.
